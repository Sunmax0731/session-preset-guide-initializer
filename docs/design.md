# 設計

## 責務分割

- input: 入力形式を配列に正規化する。
- validators: 必須項目と状態の検証だけを担当する。
- review-model: 表示用カード、完了率、次アクションを組み立てる。
- report/UI: Markdown、HTML、またはホストUIへの表示に責務を限定する。
- qcds-evaluate: 実装、ドキュメント、導入、リリース準備、競合差別化を評価する。

## UI/UX方針

- まず状態、次に不足、最後に具体的な次アクションを表示する。
- 装飾よりもスキャンしやすさを優先する。
- ユーザーが手動確認に移れるよう、docs/manual-test.md への参照を明示する。
