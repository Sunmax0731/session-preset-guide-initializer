# 評価項目・評価基準

## 評価対象

- 実装: 検証ロジック、レポート生成、UI、サンプル。
- 開発ドキュメント: 要件、仕様、設計、責務、テスト計画、リリース前計画。
- ユーザードキュメント: README、導入手順書、ユーザーガイド、手動テスト手順。
- 競合差別化: docs/competitive-benchmark.md に記載した公式標準と競合比較。

## QCDS基準

| 観点 | A以上の条件 | S帯の条件 |
| --- | --- | --- |
| Quality | 自動テスト、検証責務、テスト計画、文字化け検出が揃っている | 実装とドキュメントの両方で不足検出が自動化されている |
| Cost | 追加依存が少なく、導入手順とサンプルがある | ローカルだけで検証とドキュメントZIP化が完了する |
| Delivery | リリース手順、事前確認、ロードマップ、パッケージ手順がある | 公開前判断に必要な証跡が一式揃っている |
| Satisfaction | UI/UX方針、ユーザーガイド、手動テスト手順が読める | 競合比較にもとづく差別化と次アクションが明確である |

## グレード

- S+: 95点以上
- S-: 90点以上
- A+: 85点以上
- A-: 80点以上
- B+/B-: 70点台
- C+/C-: 60点台
- D+/D-: 60点未満
