# UI/UX ブラッシュアップ

## 方針

導入プリセット・操作ガイド・セッション初期化 は、ユーザーが「何が足りないか」と「次に何をすべきか」を一目で判断できることを優先する。

## 改善内容

- 結果だけでなく、状態ラベル、完了率、エラー数、確認数をカード表示する。
- findings には次アクションを必ず含める。
- 手動テストに進むための文書パスを画面またはレポートから辿れるようにする。
- MVP後のUIは、破壊的操作を直接実行せず、dry-run とレビューを標準にする。

## 次のUI候補

- 実データ取り込みボタン。
- レポート保存先選択。
- 手動テスト結果の追記フォーム。
- release-checklist への反映支援。
